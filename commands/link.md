# Command: /spec.link

> Install AI tool adapters and make LiveSpec commands discoverable in your project.

---

## Overview

`/spec.link [tool]`

Copies the LiveSpec command documentation into `.specs/commands/` so the AI tool can reference detailed instructions locally, and installs the appropriate adapter where each AI tool expects to find it.

**Tools:** `copilot`, `claude-code`, `cursor`, `all`

If no tool is specified, LiveSpec auto-detects which tools are present (checks for `.github/`, `CLAUDE.md`, `.cursorrules`).

---

## Steps

### Step 1 — Determine Target Tool(s)

1. If tool argument provided: use it
2. If `all` provided: process all three tools
3. If no argument:
   - Check if `.github/` directory exists → include `copilot`
   - Check if `CLAUDE.md` exists → include `claude-code`
   - Check if `.cursorrules` exists → include `cursor`
   - If nothing detected: prompt user to choose

### Step 2 — Copy Command Files to `.specs/commands/`

Copy all LiveSpec command documentation into the project so the AI can read them locally:

```
.specs/commands/
├── init.md
├── specify.md
├── plan.md
├── implement.md
├── check.md
├── explain.md
├── stack.md
└── link.md
```

- Source: `commands/*.md` from the LiveSpec repository (or `.specs/commands/` if already linked)
- Destination: `.specs/commands/` in the target project
- Skip files that already exist unless `--force` is passed

### Step 3 — Install Adapter(s)

#### For `copilot`:

1. Read `adapters/copilot/agent.md` from the LiveSpec source
2. Prepend a generated header:
   ```
   <!-- Generated by LiveSpec /spec.link — do not edit manually -->
   <!-- To update: run /spec.link copilot --force -->
   ```
3. Write to `.github/copilot-instructions.md` (create `.github/` if needed)
4. If file exists and `--force` not passed: warn and skip

#### For `claude-code`:

**Default (project-level):**
1. Read `adapters/claude-code/SKILL.md` from the LiveSpec source
2. Write to `CLAUDE.md` at the project root
3. If file exists and `--force` not passed: warn and skip

**With `--global` flag:**
1. Create symlink at `~/.claude/skills/livespec.md` → `adapters/claude-code/SKILL.md`

#### For `cursor`:

1. Read `adapters/cursor/.cursorrules` from the LiveSpec source
2. If `.cursorrules` does not exist: copy it directly
3. If `.cursorrules` already exists:
   - Check if LiveSpec section marker is present (`# LiveSpec Rules for Cursor`)
   - If marker present: replace the LiveSpec section (idempotent)
   - If marker absent and `--force` not passed: append with section separator

### Step 4 — Report

Print a summary of what was created, updated, or skipped:

```
✅ .specs/commands/ — 8 command files copied
✅ .github/copilot-instructions.md — created from adapters/copilot/agent.md
✅ CLAUDE.md — created from adapters/claude-code/SKILL.md
⚠️  .cursorrules — already exists, LiveSpec section appended
```

---

## Output

```
.specs/commands/           ← 8 command files (readable by the AI)
.github/copilot-instructions.md   ← Copilot adapter (if copilot selected)
CLAUDE.md                          ← Claude Code adapter (if claude-code selected)
.cursorrules                       ← Cursor adapter (if cursor selected)
```

---

## Flags

| Flag | Behavior |
|---|---|
| `--symlink` | Use symlinks instead of file copies (keeps commands in sync with the LiveSpec source) |
| `--global` | For Claude Code: install to `~/.claude/skills/livespec.md` instead of project root |
| `--force` | Overwrite existing adapter files without prompting |
| `--dry-run` | Show what would be created without writing any files |
| `--livespec-dir [path]` | Path to the LiveSpec repository clone (auto-detected by default) |

---

## Examples

```bash
# Auto-detect tools and link all
/spec.link

# Link only GitHub Copilot
/spec.link copilot

# Link Claude Code globally (all projects)
/spec.link claude-code --global

# Link all tools, overwriting existing files
/spec.link all --force

# Preview what would be created
/spec.link all --dry-run

# Use symlinks so command docs stay in sync
/spec.link all --symlink
```

---

## Run as a shell script

You can also run the linking script directly without an AI tool:

```bash
# From your project directory, with livespec cloned nearby:
bash /path/to/livespec/scripts/link.sh

# Specify tool explicitly:
bash /path/to/livespec/scripts/link.sh --tool copilot

# Force overwrite:
bash /path/to/livespec/scripts/link.sh --tool all --force
```
